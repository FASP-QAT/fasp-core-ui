Intro to QAT Forecasting

 

A forecast is an estimate/prediction of the quantities of products to be consumed/utilized by clients or consumers over a future period of time. In QAT, the forecast process is generally conducted in a specific order (see image below). First, set up a program using Update Program Info or select a previous program using Load or Delete Program. After loading the program, add information on the forecast period, threshold (%), products, and other program settings in the Update Version Settings and Updating Planning Unit screens. In QAT, users can either create forecasts by designing forecasting trees (tree-based forecast) or by leveraging historical consumption (consumption-based forecast) to reach a final forecast. For each product, choose consumption or tree methodology, or both!

Consumption-based forecasts are most useful in mature, stable programs that have a full supply of products and reliable data. This method can be the most reliable predictor of future consumption if future use is predicted to be very similar to the past. Ideally, 24+ months of monthly actual consumption data is available. This data can be imported directly from the QAT Supply Planning module using the Import From QAT Supply Plan screen or manually entered. Once data is added, it can be adjusted (e.g., reporting rate, stockout rate, interpolation of missing data) in the Data Entry and Adjustment screen to have a more accurate representation. In the Extrapolation screen, QAT computes future requirements by extrapolation using different forecasting methodologies (semi-averages, exponential smoothing, Holt-Winter, etc.).

If the necessary data for a consumption-based forecast is not available or not helpful in predicting future consumption, conduct a tree-based forecast (Manage Tree), which flexibly accommodates all other forecast methods, such as morbidity, services, and demographic methods. Take advantage of pre-built trees from Tree Templates or build a custom tree. Specify changes over time in a tree, either by directly specifying the rate of change or extrapolating from past data. Once a tree has been built, check the product usage using the Product Validation screen and the change over time using the Modeling Validation screen.

Once the various forecasts have been created, navigate to the Forecast Analysis and Outputs section to view forecasts, and select the final forecast. The Compare and Select Forecast screen allows comparison of all tree and consumption forecasts and selection of the final forecast per planning unit. The Monthly Forecast and Forecast Summary will provide a further breakdown or aggregation of each selected forecast. The Version Comparison screen allows comparison of selected forecasts across different program versions.

Throughout the forecasting process, save work often to the server by using the Commit Program screen. When finished with a forecast, commit forecasts as final and then import it into a Supply Plan using the QAT Forecast Import screen!

Enjoy and use the tooltips and "Show Guidance" screens throughout QAT for more guidance and tips.
