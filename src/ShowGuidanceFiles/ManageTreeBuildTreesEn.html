Manage Tree - Build Trees

Purpose Enable users to manage and build forecast tree and scenarios, for any non-consumption forecasts (demographic, morbidity, services, etc). Note that more guidance is available after clicking into any specific node.

Using this screen :
Navigation:

  • Begin by selecting the Program and Tree from the dropdowns. Note that only downloaded programs are available. Click on the gear icon to show/hide or edit the tree details (forecast method, tree name, and region).
  • See below section for Scenarios.
  • The values of the tree nodes is displayed in the date show in the Display Date. Change this date to see values in other months.
  • Users can also edit the tree in the Tree Table screen, which allows batch editing of node values, but does not allow any edits to the structure of the tree or changes over time.
  • “Hide” checkboxes - use these checkboxes to hide certain elements of the tree for a cleaner display.
  • Autocalculate - If QAT is slow while editing your tree, uncheck the auto-calculate box to stop calculations. To update the calculations, either re-check the 'auto-calculate' checkbox, or click the recalculate icon.
  • Click on the tree Levels (dark blue rectangular boxes on the left of the tree) to:
    • Name the levels of a tree. This will make a tree easier to read and will be used in the dropdowns and axis of the Modeling Validation screen.
    • Reorder branches of your tree as desired by using the up and down arrows to indicate the left-to-right position of the branches. If the list is long, use the “see children of” dropdown to filter the nodes.

Building the Tree:The forecast tree is built from the top down, using different types of "nodes". See the Node Types and Node Actions below. Each forecast tree must include one or more Planning Unit Nodes, which form the forecast output, for that tree to be available in the "Compare and Select" screen. Each Planning Unit Node must stem from a Forecasting Unit Node.

Node Actions :

  • Click on a node to open it for editing or viewing.
  • Hover over a node to view node notes without opening the node.
  • Copy / Move: Copies or moves the selected node and all its children. The user can select the destination.
  • Delete: Deletes the selected node and all its children.
  • Add Branch Template: Adds a tree template as a branch below the chosen node.
  • Add: Adds a child to the selected node.
  • / Collapse/Expand: Collapse/Expands the current node and all its children.
    • In collapsed mode, nodes will be represented by dots. Hover over any dot/node and click once to expand that node and all its children.
    • “Collapse Tree” checkbox at the top of the screen that collapses every node except for the top node.

Changes Over Time

While the tree structure stays constant throughout time, node percentages and values can change over time. Use the date dropdown to view the tree at any month. The three functionalities below are available in each node and allow the user to control how nodes change over time:

  • Modeling: Allows user to specify the exact rate of change
  • Transfer: Allows users to link two nodes together - so the decrease from the source node is linked to the increase of the destination node. Useful for transitions.
  • Extrapolation: Allows users to use historical data to extrapolate future change

Using Scenarios :
Scenarios are used to model different values for the same tree. To add, edit or delete a scenario, use the icon next to the scenario dropdown. Use the scenario dropdown to select which scenario to view and edit.
