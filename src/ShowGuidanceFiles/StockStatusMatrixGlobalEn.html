The Stock Status Matrix (Global) provides a high-level, grid-based overview of stock status (Months of Stock (MOS) or Quantity) for a single planning unit or equivalency unit across multiple programs, planning units, and time periods. To view stock status across planning units and over time within a single supply plan program, refer to the Stock Status Matrix.

For more explanation of ending balance, AMC, min/max MOS and quantity, please see the ‘Show guidance’ screen on Supply Plan Report or the User Manual (Stock Status Reports and Annex 2.)

Each planning unit can be planned by either MOS or Quantity, and their stock status is determined based on Planning Unit setting. Program admins can update the “Plan By” setting (MOS or Quantity) in the “Planning Unit Settings” screen.

Stock Status Color-coding
Cells are color-coded based on the stock status using the color coding displayed in the legend.

  • Plan by MOS: Stock status is determined by comparing the MOS with the Min MOS (user-entered in Planning Unit Settings screen) and Max MOS (calculated).
  • Plan by Quantity: Stock status is determined by comparing the quantity of the “ending balance” with the Min quantity (user-entered in Planning Unit Settings screen) and Max quantity (calculated).

Average Monthly Consumption (AMC)

*Consider only non-zero values. Also, current month is included in the months in future for AMC
If the report includes multiple programs or planning units, the AMC of the selected program/planning units will be aggregated (summed) into a single total quantity.

Months of Stock (MOS)

If the report includes multiple programs or planning units:
Months of Stock = (sum of selected programs/planning units’ ending balance / sum of selected programs/planning units’ AMC)

Example:
Program A Ending Balance = 10,000
Program A AMC = 2,500
Program B Ending Balance = 6,000
Program B AMC = 3,000
Program C Ending Balance = 750
Program C AMC = 250

Months of Stock = (10,000 + 6,000 + 750) / (2,500 + 3,000 + 250)
Months of Stock = 2.91

Min/Max Months of Stock


If the report includes multiple programs or planning units,

  • Aggregate Min MOS = the average of the selected program & planning units’ Min MOS
  • Aggregate Max MOS = the average of the selected program & planning units’ Max MOS

Example:
Program A min MOS = 3
Program A max MOS = 6
Program B min MOS = 5
Program B max MOS = 12

Aggregate Min MOS = Average (3, 5) = 4
Aggregate Max MOS = Average (6, 12) = 9

Min/Max Quantity
Min Qty = Defined by the user in program settings

Note that max quantity changes by month as AMC changes by month.

If the report includes multiple programs or planning units,

  • Aggregate Min Quantity = the average of the selected program & planning units’ Min Quantity
  • Aggregate Max Quantity = the average of the selected program & planning units’ Max Quantity

Example:
Program A min Quantity = 3,000
Program A max Quantity = 6,000
Program B min Quantity = 5,000
Program B max Quantity = 12,000

Aggregate Min Quantity = Average (3000, 5000) = 4000
Aggregate Max Quantity = Average (6000, 12000) = 9000

Show Quantity Checkbox
If unchecked (default), stock is displayed by months of stock (MOS) for planning units planned by MOS and as quantities for those planned by quantity. If checked, stock is displayed by quantity, regardless of the ‘plan by’ settings. When stock is displayed by quantity, the value represents the inventory or projected inventory available at the end of the month.

Show Icon Checkbox
If checked (default), this displays the shipment and expiry icons. There is a shipment icon displayed for every month with shipment(s), excluding cancelled, on-hold, and inactive shipments. There is an expiry icon displayed for every month with a projected expiry.

Remove Shipment Checkboxes
These checkboxes allow you to simulate different stock scenarios by excluding certain shipments (e.g. planned, unfunded) from the projected inventory

  • If both checkboxes are unchecked, all active shipments (Status = Planned, Submitted, Approved, Shipped, Arrived, Received, On-hold) are included in the projected inventory (both quantity and MOS).
  • If “Remove Planned Shipment” is checked, shipments with status = Planned are removed from the stock projections. Note: that by definition, this removes all TBD shipments since TBD shipments are always in the Planned status.
  • If “Remove TBD Funding Source Shipments” is checked, shipments with funding source = TBD are removed from the stock projections.
