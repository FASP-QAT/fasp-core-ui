A tabela de planejamento de suprimentos nesta tela usa as mesmas fórmulas da tela de Plano de Suprimentos. Para orientações sobre como o QAT calcula o saldo inicial, remessas sugeridas, estoque projetado, consumo médio mensal, meses de estoque e demanda não atendida, consulte Mostrar Orientações na tela de Planejamento de Suprimentos.

  • Cenário: Aumentar o consumo previsto

    Percentual inserido pelo usuário durante o período selecionado. Aplica-se apenas a meses com valores previstos e sem consumo real.

    Exemplo:
    Aumentar a previsão em 10% de janeiro a março
    Consumo previsto: Jan = 100, Fev = 110, Mar = 112
    Consumo previsto aumentado: Jan = 100 x 1.1, Fev = 110 x 1.1, Mar = 112 x 1.1
    Consumo previsto aumentado: Jan = 110, Fev = 121, Mar = 123

  • Cenário: Reduzir o consumo previsto

    Percentual inserido pelo usuário durante o período selecionado. Aplica-se apenas a meses com valores previstos e sem consumo real.

    Exemplo:
    Reduzir a previsão em 10% de janeiro a março
    Consumo previsto: Jan = 100, Fev = 110, Mar = 112
    Consumo previsto reduzido: Jan = 100 x 0.9, Fev = 110 x 0.9, Mar = 112 x 0.9
    Consumo previsto reduzido: Jan = 90, Fev = 99, Mar = 101

  • Cenário: Remover remessas sem financiamento

    O QAT remove todas as remessas em um período selecionado pelo usuário com fonte de financiamento = A Definir (TBD), e portanto em status Planejado.

  • Cenário: Remover remessas planejadas

    O QAT remove todas as remessas com status = Planejado.
    O usuário pode optar por remover apenas as remessas que não cumprem o prazo de entrega conforme a lógica do programa.

    Exemplo:

    Remover todas as remessas planejadas
    Mês atual = Jan
    Prazo de entrega = 8 meses
    Remessas planejadas para Fev, Abr, Jul, Out e Nov.
    O novo plano de suprimentos remove todas as cinco remessas.

    Remover remessas que não cumprem o prazo
    Novamente, remessas para Fev, Abr, Jul, Out e Nov.
    Novo plano remove remessas entre Jan e Set (Jan + 8 meses).
    O novo plano mantém Out e Nov.

  • Cenário: Remover todas as remessas enviadas que não cumprem o prazo

    O QAT remove todas as remessas com status = Enviado (Submitted) que não cumprem o prazo com base na lógica do programa.

    Exemplo:

    Mês atual = Jan
    Prazo entre Enviado e Recebido = 4 meses
    Remessas com datas de recebimento em Fev, Abr, Jul, Out e Nov.
    Novo plano remove remessas de Jan até Mai (Jan + 4 meses).
    Mantém Jul, Out e Nov.

  • Cenário: Remover todas as remessas em espera que não cumprem o prazo

    O QAT remove todas as remessas com status = Em Espera (On-hold) que não cumprem o prazo de entrega com base na lógica do programa.

  • Cenário: Replanejar plano de suprimentos

    Para um período inserido pelo usuário, o QAT remove todas as remessas Planejadas dos agentes de compras e fontes de financiamento selecionados pelo usuário; em seguida, usa a lógica de remessas sugeridas para criar novas remessas que atendam aos requisitos de tempo de entrega e MIN/MÁX.

    Lógica das Remessas Sugeridas - “Planejar por MOS”

    Nota: Abaixo está a lógica para unidades de planejamento que são planejadas por meses de estoque (MOS). Atualize a configuração “planejar por” na tela Atualizar unidade de planejamento.

    Onde Mês N é o mês para o qual o QAT está calculando:

    1. Se AMC = 0 ou N/A para o Mês N, nenhuma remessa sugerida
    2. Se o Mês N estiver sem estoque (Saldo Final = 0), o QAT sempre sugerirá uma remessa
      1. Se ambos os próximos 2 meses (N+1, N+2) forem < Min MOS, a remessa sugerida levará o mês N a Max MOS. Quantidade sugerida = (Max MOS para N) * (AMC para N) – (Saldo Final para N) + (Demanda Não Atendida para N)
      2. Se 1 ou ambos os próximos 2 meses (N+1 ou N+2) forem > Min MOS, a remessa sugerida levará o mês N a Min MOS. Quantidade sugerida = (Min MOS para N) * (AMC para N) – (Saldo Final para N) + (Demanda Não Atendida para N)
    3. Se o produto estiver com estoque insuficiente (MOS < Min MOS) por 3 meses consecutivos (Mês N, N+1, N+2)?
      1. Se sim, a remessa sugerida levará o Mês N a Max MOS. Quantidade sugerida = (Max MOS para N) * (AMC para N) – (Saldo Final para N)
      2. Se não, nenhuma remessa sugerida
    Remessas Sugeridas - “Planejar por Quantidade”

    Nota: Abaixo está a lógica para unidades de planejamento que são planejadas por meses de estoque (MOS). Atualize a configuração “planejar por” na tela Atualizar unidade de planejamento.

    Onde Mês N é o mês para o qual o QAT está calculando, e X = Tempo de Entrega da Distribuição:

    1. Se AMC = 0 ou N/A para o Mês N, nenhuma remessa sugerida
    2. Se o Mês N estiver sem estoque (Saldo Final = 0), o QAT sempre sugerirá uma remessa
      1. Se ambos os próximos 2 meses (N+1, N+2) forem < Min Qty, a remessa sugerida é a quantidade que levaria o Mês N a Max Qty. Quantidade sugerida = (Max Qty para N) – (Saldo Final para N) + (Demanda Não Atendida para N)
      2. Se um ou ambos os próximos 2 meses (N+1 ou N+2) forem > Min Qty, a remessa sugerida é a quantidade que levaria o Mês N a Min Qty. Quantidade sugerida = (Min Qty) – (Saldo Final para N) + (Demanda Não Atendida para N)
    3. Se o produto estiver com estoque insuficiente (Saldo Final < Min Qty) por 3 meses consecutivos (Mês N, N+1, N+2)?
      1. Se sim, a remessa sugerida é a quantidade que levaria o Mês N a Max Qty. Quantidade sugerida = (Max Qty para N) – (Saldo Final para N)
      2. Se não, nenhuma remessa sugerida

    O QAT coloca a quantidade sugerida no Mês N–X. Em outras palavras, X meses antes do Mês N, onde X é o Tempo de Entrega da Distribuição.
