La tabla de planificación de suministros en esta pantalla utiliza las mismas fórmulas que en la pantalla de Plan de Suministro. Para obtener orientación sobre cómo QAT calcula el saldo inicial, los envíos sugeridos, el inventario proyectado, el consumo mensual promedio, los meses de inventario y la demanda no satisfecha, consulte la opción Mostrar Guía en la pantalla de Planificación de Suministros.

  • Escenario: Aumentar el consumo pronosticado

    Porcentaje ingresado por el usuario durante el período seleccionado. Solo se aplica a meses con valores pronosticados y sin consumo real.

    Ejemplo:
    Aumentar el pronóstico en un 10% de enero a marzo
    Consumo pronosticado: Ene = 100, Feb = 110, Mar = 112
    Consumo pronosticado aumentado: Ene = 100 x 1.1, Feb = 110 x 1.1, Mar = 112 x 1.1
    Resultado: Ene = 110, Feb = 121, Mar = 123

  • Escenario: Disminuir el consumo pronosticado

    Porcentaje ingresado por el usuario durante el período seleccionado. Solo se aplica a meses con valores pronosticados y sin consumo real.

    Ejemplo:
    Disminuir el pronóstico en un 10% de enero a marzo
    Consumo pronosticado: Ene = 100, Feb = 110, Mar = 112
    Consumo pronosticado disminuido: Ene = 100 x 0.9, Feb = 110 x 0.9, Mar = 112 x 0.9
    Resultado: Ene = 90, Feb = 99, Mar = 101

  • Escenario: Introducción/eliminación progresiva del pronóstico

    Período seleccionado por el usuario, valor inicial y valor final. QAT interpola los nuevos valores de pronóstico entre los valores de inicio y fin utilizando una fórmula de línea recta.

    Matemáticamente:

    • Dónde x representa los meses y y representa el consumo pronosticado,
    • Datos conocidos: (x0, y0) y (x1, y1)
    • Valores desconocidos: (x, y)
    • Fórmula de interpolación lineal:

    Solo se aplica a meses con consumo pronosticado y sin consumo real. Los valores después del período seleccionado no se modifican.

    Ejemplo:
    Eliminación gradual del producto de julio a diciembre (6 meses).
    Consumo actual: Jul = 200, Ago = 185, Sep = 177, Oct = 160, Nov = 145, Dic = 150
    Valor inicial Jul = 200 (auto poblado por QAT; editable)
    Valor final Dic = 0 (ingresado por el usuario)
    Nuevo pronóstico:
    Jul = 200,
    Ago = 160,
    Sep = 120,
    Oct = 80,
    Nov = 40,
    Dic = 0

  • Escenario: Eliminar envíos no financiados

    QAT elimina todos los envíos en un período seleccionado por el usuario con fuente de financiación = TBD y estado = Planificado.

  • Escenario: Eliminar envíos planificados

    QAT elimina todos los envíos con estado = Planificado.
    El usuario puede optar por eliminar solo los envíos planificados que no cumplan con el tiempo de entrega según la lógica del programa.

    Ejemplo:

    Eliminar todos los envíos planificados
    Mes actual = Ene
    Tiempo de entrega = 8 meses
    Envíos planificados: Feb, Abr, Jul, Oct, Nov
    El nuevo plan elimina los cinco envíos.

    Eliminar envíos que no cumplen con el tiempo de entrega
    Envíos planificados: Feb, Abr, Jul, Oct, Nov
    El nuevo plan elimina envíos entre Ene y Sep (Ene + 8 meses).
    El nuevo plan conserva los envíos de Oct y Nov.

  • Escenario: Eliminar todos los envíos Enviados que no cumplen con el tiempo de entrega

    QAT elimina todos los envíos con estado = Enviado que no cumplen con el tiempo de entrega según la lógica del programa.

    Ejemplo:

    Mes actual = Ene
    Tiempo de entrega desde Enviado hasta Recibido = 4 meses
    Envíos enviados con fechas de recepción en Feb, Abr, Jul, Oct, Nov
    El nuevo plan elimina envíos entre Ene y May (Ene + 4 meses)
    El nuevo plan conserva los envíos de Jul, Oct, y Nov.

  • Escenario: Eliminar todos los envíos En espera que no cumplen con el tiempo de entrega

    QAT elimina todos los envíos con estado = En espera que no cumplen con el tiempo de entrega según la lógica del programa.

  • Escenario: Replanificar el plan de suministro

    Para un período ingresado por el usuario, QAT elimina todos los envíos planificados de los agentes de adquisiciones y fuentes de financiamiento seleccionados por el usuario; luego utiliza la lógica de envíos sugeridos para crear nuevos envíos que cumplan con los requisitos de tiempo de entrega y MIN/MÁX.

    Lógica de envíos sugeridos - “Planificar por MES”

    Nota: A continuación se muestra la lógica para las unidades de planificación que se planifican por meses de inventario (MOS). Actualice la configuración de “planificar por” en la pantalla Actualizar unidad de planificación.

    Donde el Mes N es el mes para el que QAT está calculando:

    1. Si el CMM = 0 o N/D para el Mes N, no hay envío sugerido
    2. Si el Mes N está agotado (Saldo Final = 0), QAT siempre sugerirá un envío
      1. Si ambos de los siguientes 2 meses (N+1, N+2) están < MOS Mínimo, el envío sugerido llevará el mes N a MOS Máximo. Cantidad sugerida = (MOS Máx para N) * (CMM para N) – (Saldo Final para N) + (Demanda no satisfecha para N)
      2. Si 1 o ambos de los próximos 2 meses (N+1 o N+2) están > MOS Mínimo, el envío sugerido llevará el mes N a MOS Mínimo. Cantidad sugerida = (MOS Mín para N) * (CMM para N) – (Saldo Final para N) + (Demanda no satisfecha para N)
    3. ¿El producto está con inventario insuficiente (MOS < MOS Mínimo) durante 3 meses consecutivos (Mes N, N+1, N+2)?
      1. Si es así, el envío sugerido llevará el Mes N a MOS Máximo. Cantidad sugerida = (MOS Máx para N) * (CMM para N) – (Saldo Final para N)
      2. Si no, no hay envío sugerido
    Envíos sugeridos - “Planificar por Cantidad”

    Nota: A continuación se muestra la lógica para las unidades de planificación que se planifican por meses de inventario (MOS). Actualice la configuración de “planificar por” en la pantalla Actualizar unidad de planificación.

    Donde el Mes N es el mes que QAT está calculando, y X = Tiempo de Entrega de Distribución:

    1. Si el CMM = 0 o N/D para el Mes N, no hay envío sugerido
    2. Si el Mes N está agotado (Saldo Final = 0), QAT siempre sugerirá un envío
      1. Si ambos de los siguientes 2 meses (N+1, N+2) están < Cantidad Mínima, el envío sugerido será la cantidad que lleve el Mes N a Cantidad Máxima. Cantidad Sugerida = (Cant. Máx para N) – (Saldo Final para N) + (Demanda no satisfecha para N)
      2. Si uno o ambos de los próximos 2 meses (N+1 o N+2) están > Cantidad Mínima, el envío sugerido será la cantidad que lleve el Mes N a Cantidad Mínima. Cantidad Sugerida = (Cant. Mínima) – (Saldo Final para N) + (Demanda no satisfecha para N)
    3. ¿El producto tiene inventario insuficiente (Saldo Final < Cantidad Mínima) durante 3 meses consecutivos (Mes N, N+1, N+2)?
      1. Si es así, el envío sugerido es la cantidad que lleve el Mes N a Cantidad Máxima. Cantidad Sugerida = (Cant. Máx para N) – (Saldo Final para N)
      2. Si no, no hay envío sugerido

    QAT coloca la cantidad sugerida en el Mes N–X. Es decir, X meses antes del Mes N, donde X es el Tiempo de Entrega de Distribución.
