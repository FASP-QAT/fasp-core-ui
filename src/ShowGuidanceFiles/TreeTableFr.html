Tips for specific use cases:

  • Type de nœud de prévision : Sélectionnez discret si ce produit sera utilisé pendant une durée limitée et continue. s'il sera utilisé indéfiniment.
  • Utilisation retardée ou échelonnée du produit ? Parfois, la consommation du produit est retardée par rapport à l'autre plus élevée. niveaux de l'arbre. Dans le nœud Unité de prévision approprié, utilisez le champ Décalage pour signaler ce retard.
    • Par exemple, si l'utilisation du produit commence 2 mois après les dates du nœud parent, entrez 2 dans ce champ.
    • Ce champ peut également être utilisé lorsque le produit change au fil du temps, par exemple : si les unités de prévision A, B et C sont utilisées en sécession pendant deux mois à la fois, vous pouvez paramétrer votre arbre avec l'Unité de Prévision A (discrète sur 2 mois, lag=0), Unité de prévision B (discrète sur 2 mois, décalage = 2), unité de prévision C (discret pendant 2 mois, décalage = 4).
    • Repeating Forecasting Usages? If multiple Forecasting Unit nodes share the same settings, consider using the Usage Templates screen to save your common usages, and then using the “Copy from Template” field to populate the fields in the forecasting unit nodes.
